## What Is Collected

- All prompts and instructions submitted to v0
- Generated code and component output
- User account information (linked to Vercel account)
- Session and usage analytics
- Browser and device information
- Iteration history (all revisions of a generation)
- Sharing/publishing events
- Error reports and logs
- Feature usage metrics

## What Is NOT Collected

- Local file system contents (v0 is a web-based tool)
- IDE or local development environment data

## How to Fully Opt Out

1. **Vercel CLI telemetry** (if using Vercel CLI alongside v0):
   ```bash
   vercel telemetry disable
   ```
   Note: This only affects the Vercel CLI, not the v0 web service.

2. **v0 web service:** There is no documented opt-out for analytics on the v0 web platform. Data collection is integral to the SaaS service.

3. The only way to fully avoid data collection is to not use the service.

## Red Flags

- All prompts and generated code are processed and stored on Vercel's servers
- No granular opt-out for the web-based v0 service
- Closed source: no way to verify data handling practices through code review
- Generated outputs may be used to improve the service (check terms of service)
- Prompts may contain proprietary design specifications or business logic descriptions
