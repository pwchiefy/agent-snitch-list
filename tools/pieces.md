## What Is Collected

- Usage analytics and feature engagement metrics
- Application performance data
- OS, version, and hardware info
- Account data (if cloud features enabled)
- When cloud features are enabled: code snippets and context may be processed server-side

## What Is NOT Collected

- Code stays on-device when using local/offline mode
- Snippets are not sent to Pieces servers in default local mode
- No code training on user data (per Pieces privacy claims)

## How to Fully Opt Out

1. Open Pieces Desktop App
2. Navigate to Settings
3. Disable analytics/telemetry checkbox
4. Use only local/offline LLM mode to ensure code never leaves device

## Red Flags

- Closed source -- cannot independently verify on-device claims
- Cloud mode sends code to Pieces servers for processing
- Distinction between "local" and "cloud" mode may confuse users about what data leaves the device

## Evidence

- Privacy policy: https://pieces.app/privacy
- Settings UI contains telemetry toggles
- Marketing emphasizes on-device processing as a differentiator
